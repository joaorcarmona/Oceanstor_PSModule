# Huawei OceanStor Report Templates

The XML files in this folder define which object properties are included in
Excel reports generated by POSH-Oceanstor.

The templates are used by `Export-DMStorageToExcel` when it builds report
worksheets for these report types:

- `disks`
- `hosts`
- `hostgroups`
- `lungroups`
- `lunsv3`
- `lunsv6`

Other exported worksheets, such as `system`, `storagepools`, and `vstores`, are
written directly from the collected objects and do not currently use these XML
templates.

## How Templates Work

Each template has a root element that must match the report type. For example,
`Report-Disks.xml` uses `<disks>`, and the LUN templates use `<lunsv3>` or
`<lunsv6>`.

Inside the root element, each `<property>` entry controls one exported column:

```xml
<property>
  <name>id</name>
  <order>1</order>
  <description></description>
  <enabled>1</enabled>
</property>
```

The fields mean:

- `name`: The object property to export. This must match the property name on
  the object returned by the module, so change it only when you know the object
  exposes a different property.
- `order`: The column order in the generated report. Lower numbers are exported
  first.
- `description`: A human-readable note about the property. This is useful for
  documenting the template, but it is not exported as a report column.
- `enabled`: Set to `1` to include the property, or `0` to exclude it.

If two properties use the same `order` value, both are exported. Their relative
order follows the order in the XML file.

## Default Template Variables

When the module is imported, `Private/Import-ReportTemplates.ps1` defines global
variables that point to the default templates:

```powershell
$global:Lunsv3ReportTemplate    = $workDir + "/Templates/Report-Lunsv3.xml"
$global:Lunsv6ReportTemplate    = $workDir + "/Templates/Report-Lunsv6.xml"
$global:HostsReportTemplate     = $workDir + "/Templates/Report-Hosts.xml"
$global:HostGroupsReportTemplate = $workDir + "/Templates/Report-HostGroups.xml"
$global:LunGroupsReportTemplate = $workDir + "/Templates/Report-LunGroups.xml"
$global:DisksReportTemplate     = $workDir + "/Templates/Report-Disks.xml"
```

`Export-DMStorageToExcel` uses those variables internally when it creates the
template-driven worksheets.

## Customizing Reports

You can edit the default XML files directly, but it is safer to copy a template
and customize the copy. That keeps the original template available as a backup
and avoids losing changes when the module is updated.

Example: copy the disks template, disable a column, and use the custom template
for a report.

```powershell
$templateRoot = Join-Path (Split-Path (Get-Module POSH-Oceanstor).Path) 'Templates'

Copy-Item `
    -Path (Join-Path $templateRoot 'Report-Disks.xml') `
    -Destination "$HOME/Report-Disks.Custom.xml"

# Edit $HOME/Report-Disks.Custom.xml, then point the report variable to it.
$global:DisksReportTemplate = "$HOME/Report-Disks.Custom.xml"

$storage = Export-DeviceManager -Hostname 10.0.0.1
Export-DMStorageToExcel -OceanStor $storage -ReportFile C:\Temp\Storage.xlsx -IncludeObject disks
```

### Disabling A Property

Change `enabled` from `1` to `0`.

Before:

```xml
<property>
  <name>id</name>
  <order>1</order>
  <description></description>
  <enabled>1</enabled>
</property>
```

After:

```xml
<property>
  <name>id</name>
  <order>1</order>
  <description></description>
  <enabled>0</enabled>
</property>
```

### Changing Column Order

Change the `order` value.

Before:

```xml
<property>
  <name>Running Status</name>
  <order>6</order>
  <description></description>
  <enabled>1</enabled>
</property>
<property>
  <name>Enclosure ID</name>
  <order>7</order>
  <description></description>
  <enabled>1</enabled>
</property>
```

After:

```xml
<property>
  <name>Running Status</name>
  <order>8</order>
  <description></description>
  <enabled>1</enabled>
</property>
<property>
  <name>Enclosure ID</name>
  <order>7</order>
  <description></description>
  <enabled>1</enabled>
</property>
```

## Notes

- The template `name` values are case-sensitive when accessed as PowerShell
  object properties. Match the property names returned by the module.
- The report type and XML root element must match. A `disks` report must use a
  `<disks>` template, a `hosts` report must use a `<hosts>` template, and so on.
- `Export-DMStorageToExcel` requires the ImportExcel module because it writes
  Excel workbooks.
- On case-sensitive filesystems, make sure the LUN V6 template filename matches
  `Report-Lunsv6.xml`, which is the path used by the module variable.
